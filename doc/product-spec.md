# 产品规格

## 定位

本应用是 HarmonyOS 上的 DeepSeek Chat Client。

目标用户是想在手机、平板或 HarmonyOS 设备上舒服使用 DeepSeek 的人。他们需要一个稳定、顺手、没有多余表演的 AI 对话应用。

## 核心体验

必须优先做好：

- API Key 设置。
- API Key 连通性检查。
- DeepSeek Chat 对话。
- DeepSeek Reasoner 对话。
- 联网搜索。
- 流式输出。
- 停止生成。
- 重新生成。
- 复制回复。
- 复制代码块。
- Markdown 渲染。
- 会话列表。
- 新建会话。
- 切换会话。
- 重命名会话。
- 删除会话。
- 消息删除和编辑后重发。
- Reasoning 折叠展示。
- 消息详情中展示模型、token、结束原因。
- 前后台切换不丢状态。
- 网络错误、限流、鉴权失败提示清楚。
- 搜索失败时能退化为普通对话。

## 可选增强

可以做，但不能抢主线：

- 图片 OCR：用户明确选择图片并要求识别时，把识别文本放入输入框或附件上下文。
- 语音输入：用户点击麦克风时转写到输入框。
- 分享文本进入应用：分享文本或链接后填入输入框。
- 余额查询：设置页或菜单里手动触发。
- 模型列表诊断：设置页手动刷新，确认 `deepseek-chat` 和 `deepseek-reasoner` 可用。
- Token 统计：设置页展示轻量使用统计。

## 非目标

不做：

- 系统 AI 助手。
- 生活工具合集。
- 通用扩展平台。
- 外部协议客户端。
- 写代码代理。
- shell / terminal / git / 构建 / 测试执行。
- 自动任务规划。
- 自动工具循环。
- 复杂消息卡片。

## 产品原则

### 1. 对话优先

用户打开应用后，最重要的是能立刻开始对话。任何设置、工具、附件、菜单都不能压过输入框和消息流。

### 2. 消息流纯净

消息流只展示：

- 用户文本。
- Assistant 文本。
- Reasoning 折叠块。
- Markdown。
- 代码块。
- 少量单行动作。

不要展示：

- 大卡片。
- 工具日志。
- 技能诊断。
- 路由信息。
- 内部状态机。
- token 表格。
- 响应 ID。
- 余额信息。

这些信息可以进入消息详情或设置页，不能进入主消息流。

### 3. 工具明确触发

默认不调工具。

只有用户打开“联网”开关，或明确说“搜索”“查一下最新”“联网查”“识别图片”“语音输入”“查余额”时才调用对应能力。

联网搜索是输入增强，不是工具循环。搜索结果只作为 DeepSeek 的上下文，不在消息流里展示大卡片；回复可以用 `[1]`、`[2]` 引用来源，来源列表放在消息详情里。

DeepSeek 服务器返回的 usage、finish_reason、模型信息应自动保存，但默认不打扰用户。只有发生截断、鉴权失败、余额不足等需要用户知道的状态，才在 UI 上提示。

### 4. 平台边界真实

HarmonyOS App 有沙箱边界。不要假装可以任意读写用户文件，不要假装有可靠 terminal。

### 5. 公开项目要干净

README、提交记录、代码结构都应该让人一眼看懂：这是一个 DeepSeek Chat Client。

## MVP 范围

第一版只做：

- 首次启动引导设置 API Key。
- 主对话页。
- 会话侧边栏。
- 设置页。
- DeepSeek chat / reasoner 切换。
- 联网搜索开关。
- 流式输出。
- Markdown / 代码块。
- 本地持久化。
- 主题和背景设置。
- 消息元数据和 usage 持久化。
- 设置页模型可用性检查。

第二版再做：

- OCR。
- 语音输入。
- 分享入口。
- 余额查询。
- Token 统计面板。

不要在第一版做工具系统。
